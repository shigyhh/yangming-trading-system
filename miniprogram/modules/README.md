# 小程序修行闭环模块

小程序不是金融工具入口，而是交易者每天进入的修行空间。所有页面都围绕这个闭环生长：

人格照见 -> 主修关卡 -> 每日事上练 -> 知行指数

## 模块边界

- `mind`：开盘照心，照见当下心境，生成今日戒律、禁忌、只练一事、仓位边界和知行提醒。
- `personality`：九型交易人格，负责照见、画像、惯性动作、心学解释和对应事上练方向。
- `practice`：今日修行，每天只练一件事，形成知行值。
- `review`：交易省察，做心性省察，不做传统行情回看。
- `index`：知行指数，聚合纪律、情绪、系统一致性、临盘克制和总分。
- `stages`：六大修行关卡，建立立志、照心、事上练、破心中贼、知行合一、致良知的主轴。
- `content365`：365天修行内容库，沉淀每日心证、戒律、事上练、省察，供小程序、直播、视频号和AI观心教练复用。
- `growth`：修行成长树，聚合连续修行和关卡进度，让用户看见自己正在往哪一关修。
- `zhixing`：知行指数、历史记录、周趋势、月趋势和修行等级。
- `cultivation`：V1成长引擎，统一输出人格映射、主修关卡、每日事上练卡、知行指数和当前修行快照。
- `coach`：AI观心教练预留接口，暂不接真实模型，先输出触发人格、当前心贼、关卡建议、今日只练。
- `dojo`：修行道场，聚合阳明先生道统导师、修行营教练/助教真人带练、AI观心助手、人格心证入口和本地修行榜。
- `profile`：修行体系、境界、积分、连续打卡和个人状态。
- `companion`：每日陪伴，不做提醒轰炸，只给一句稳心提示。

## 每日修行工具属性

第一阶段必须适合每天使用：

- 打卡：留下今天是否完成修行。
- 照心：开盘前识别当下心境。
- 照见：阶段性照见人格惯性。
- 每日任务：一天只事上练一件事。
- 知行指数：看见今天是否知行合一。
- 省察：收盘后做心性省察。
- 陪伴：每天一句稳心提醒。

## 产品原则

不要把这个产品做成金融工具，而是做成交易者每天进入的修行空间。

我们不是帮助用户预测市场，而是帮助交易者在市场中照见自己。

## V1 固化边界

暂停新增页面。当前只固化：

- 六大关卡体系。
- 九型人格与关卡映射。
- 知行指数算法。
- 365天心证库数据结构。
- 每日事上练卡数据结构。
- AI观心教练接口预留。

核心运行方式：

人格照见 -> 主修关卡 -> 每日事上练 -> 知行指数。

页面不能各自决定成长逻辑，必须优先复用 `cultivation`、`personality-stage-map`、`stages`、`practice`、`zhixing` 这些规则模块。
