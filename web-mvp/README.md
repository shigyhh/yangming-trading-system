# 九种交易人格 AI 测评网页 MVP

这是一个可本地运行的网页 MVP，用于验证“抽题-作答-评分-报告-接口提交”的完整闭环。

## 当前已实现

- 正式模式从后端随机抽取题目，不在前端暴露完整 3600 道题库。
- 支持 27 题、45 题、90 题三种测评模式。
- 每次按 9 种人格分层随机抽题，尽量覆盖不同子维度。
- 用户按 1-5 分作答后，自动计算主人格、副人格、风险等级、高分子维度和训练营分层。
- 本地生成《九种交易人格 AI 诊断报告》。
- 支持复制报告、导出 JSON。
- 增加登录门、微信绑定入口、手机号/邀请码体验入口和助理二维码承接。
- 首页增加“影响 1 亿股民建立稳定交易人格”愿景区、报告份数、注册人数、测评人数和指数图。
- 预留 Coze 工作流 B 调用入口。
- 预留飞书 Webhook 发送入口。
- 增加 `robots.txt` 和 `noindex`，避免 MVP 阶段被搜索引擎收录。

## 本地启动

在 `web-mvp` 目录下运行：

```bash
python3 -m http.server 4173
```

浏览器打开：

```text
http://localhost:4173/
```

不要直接双击 `index.html` 打开，因为浏览器会限制本地 JSON 读取。

## Coze 对接方式

页面会生成如下结构的 payload：

```json
{
  "nickname": "测试用户",
  "test_version": "45",
  "channel": "直播间",
  "answers": [
    {
      "question_id": "CX001",
      "personality_type": "冲动型",
      "sub_dimension": "追涨冲动",
      "score": 5,
      "weight": 1
    }
  ],
  "local_score_result": {}
}
```

如果 Coze 工作流 B 的接口接收字段仍然是 `input`，网页会按这个格式提交：

```json
{
  "input": "{\"nickname\":\"测试用户\",\"test_version\":\"45\",\"answers\":[...]}"
}
```

Coze 返回字段建议使用：

```json
{
  "report": "最终诊断报告正文"
}
```

如果返回的是 `output`，页面也能识别。

## 飞书对接方式

飞书机器人 Webhook 可以先用于内部通知，发送内容包括昵称、主人格、副人格、风险等级和训练营建议。

正式版建议用后端中转，不要把长期有效的敏感 token 暴露在前端页面里。

## 下一步建议

1. 先用本地 MVP 跑 20-50 个真实用户，观察题目是否容易理解、结果是否被认可。
2. 把 Coze 报告提示词再根据真实反馈收紧，形成“客户觉得准”的话术。
3. 增加用户手机号或企微线索字段，接飞书多维表格沉淀数据。
4. 再升级成正式版：登录、数据库、后台查看、批量导出、训练营推荐页。

安全和防爬虫方案见 `SECURITY.md`。注意：当前静态登录门只用于演示，正式上线必须把题库、评分和报告生成迁移到服务端。
