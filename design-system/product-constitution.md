# 阳明心学交易系统小程序产品宪法

## 一句话

不要把这个产品做成金融工具，而是做成交易者每天进入的修行空间。

我们不是帮助用户预测市场，而是帮助交易者在市场中照见自己。

## 产品本质

这个小程序不是行情软件、选股工具、社区论坛，也不是投顾系统。它是交易人格修行系统。

它训练的不是预测能力，而是：

- 计划能不能被执行
- 情绪能不能被照见
- 系统能不能被守住
- 错误能不能被省察
- 明日能不能只修正一件事

## 修行闭环

所有功能必须服务于这个闭环：

人格识别 -> 主修关卡 -> 每日训练 -> 知行指数

长期成长闭环：

立志 -> 照心 -> 事上磨 -> 破心贼 -> 知行合一 -> 致良知

任何新增功能必须回答：

- 它在闭环里的位置是什么？
- 它训练了用户哪一个心性动作？
- 它有没有诱导用户把注意力转向结果、行情或外部刺激？

## V1 固化边界

V1 不新增无关页面，先固化以下六个系统规则：

1. 六大关卡体系。
2. 九型人格与关卡映射。
3. 知行指数算法。
4. 365天心证库数据结构。
5. 每日训练卡数据结构。
6. AI观心教练接口预留。

## 六个核心模块

1. 开盘照心：开盘前识别心境，生成今日戒律、禁忌、只练一事、仓位边界、知行提醒。
2. 九型人格：识别主导交易人格，用于传播和训练分层。
3. 今日修行：每天只练一件事，形成打卡和知行值。
4. 交易复盘：做心性复盘，不做传统行情复盘，生成AI省察与AI复盘。
5. 知行指数：以纪律、情绪、系统一致性、临盘克制构成总分。
6. 修行体系：修行成长树与六大关卡进度，让用户看见自己正在往哪一关修。
7. 修行道场：阳明先生作为道统导师，训练营教练/助教作为真人带练；AI观心助手使用照心、人格档案、训练、复盘、成长树和连续修行上下文，只做观心追问和稳心提醒。

其中九型人格必须映射六大关卡，知行指数必须绑定人格、主修关卡、每日训练和省察记录；页面只能呈现成长体系结果，不能变成互不关联的功能入口。

## 六大关卡体系

1. 立志：核心是交易戒律建立；训练无计划不开仓、仓位先于交易。
2. 照心：核心是识别情绪；训练贪、惧、急、怒、疲。
3. 事上磨：核心是盘中执行；训练止损执行、不追涨、不扛单。
4. 破心贼：核心是克服惯性人格；训练赌徒型、冲动型、扛单型。
5. 知行合一：核心是系统执行一致性；训练连续执行、不临盘改计划。
6. 致良知：核心是稳定成熟交易者；训练自主复盘、自我觉察。

## 开发顺序

世界观 -> 训练体系 -> 用户验证 -> AI观心教练 -> 后台。

不要先做后台和复杂运营工具。后台只在训练体系和用户闭环验证后再放大。

## 每日修行工具属性

小程序每天必须可用，适合：

- 打卡
- 照心
- 测评
- 每日任务
- 知行指数
- 复盘
- 陪伴

其中“陪伴”不是强销售触达，而是低频、克制、稳定的修行提醒。

## 文案准则

强化：

- 知行合一
- 照心
- 立志
- 戒律
- 事上磨
- 致良知
- 省察
- 破心贼
- 观心
- 克己
- 修行
- 心境

禁止：

- 暴富
- 翻倍
- 妖股
- 龙头
- 必赢
- 稳赚
- 财富自由

禁止展示任何具体股票代码和结果截图。

## 视觉准则

- 黑金
- 东方
- 克制
- 留白
- 呼吸感
- 黑里还有黑
- 金色只做方向和仪式感
- 主模块要像法旨，不像信息卡

## 动效准则

- 慢
- 柔
- 呼吸
- 景深
- 水波纹
- 不做强刺激
- 不做销售感

## 合规边界

系统只能表达交易认知、行为训练与风险教育。不能输出具体买卖指令，不能承诺结果，不能展示具体股票代码或结果截图。
