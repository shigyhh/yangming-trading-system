# 07 Brand Usage Boundary

这一章解决一个问题：标识系统不是越多越有品牌感，而是每一次出现都要有位置、有理由、有边界。

当前阶段只建立最小可用标识系统，服务 Sprint 8-10 的首页、测评入口、报告卡、训练记录、复测变化、用户中心和数据绑定。长卷模式、复杂社区、全球照见层正式视觉暂不展开。

## 标识角色

| 资产 | 角色 | 主要用途 | 不承担 |
| --- | --- | --- | --- |
| A1 主标 | 正式品牌落款 | 官网导航、报告落款、课程封面、正式分享封面 | 小尺寸按钮、重复装饰、页面背景纹样 |
| C16 小标 | 小尺寸记忆点 | favicon、小程序头像、移动端角标、24px 以下 UI | 完整读字、海报主视觉、复杂叙事 |
| UI Glyph | 功能母题 | 交易、复盘、训练、成长四类入口和卡片 | Logo 替代、交易信号、行情指标 |
| 文字锁定 | 品牌识别补充 | 导航、封面、落款、分享卡标题区 | 同屏重复、正文强调、按钮文案 |

## 单屏规则

- 一个首屏只允许一个品牌锁定。
- 导航已经出现 A1 + 产品名时，首屏正文区域不再重复 A1 或“阳明照见系统”。
- 同一个卡片内最多出现一个 Glyph。
- 同一屏 Glyph 数量超过四个时，必须降级为文字标签或列表，不继续堆符号。
- 品牌符号不能比用户任务更抢眼。首页第一优先级永远是“心不静，交易必乱。”。

## 跨端落位

### Web 首页

- 顶部导航使用 A1 主标 + 产品名。
- 首屏主标题不重复品牌锁定。
- AI 观心卡可以少量使用 UI Glyph，作为“交易、复盘、训练、成长”的母题提醒。
- 未来长卷入口未上线前，不在首页加入完整闭环卡片。

### Web 报告页

- 报告卡头部可以使用 A1 主标，表示正式报告落款。
- 风险雷达、七日训练、复盘证据使用 UI Glyph 时，必须配合文字标题。
- 不用 Glyph 表达分数涨跌、收益暗示或任何交易方向。

### Web 训练页

- 使用 C16 或训练 Glyph 作为小型识别。
- 训练动作优先展示文字，符号只辅助识别。
- “停十秒、先照心、再行动”这类动作文案不能被图形替代。

### 小程序

- 小程序头像、首页小标、报告页小标使用 CSS 几何 C16。
- 小程序不主动引用未确认授权字体。
- 品牌标必须有替换文字，例如 `aria-label="阳明照见 C16 小标"`。
- 截图预览必须有替换文字，说明这是“交易复盘截图预览”或“交易复盘截图详情”。

### 分享卡

- 分享卡只允许一个主标或一个小标。
- 若画面已有大标题和人格报告，不再额外加入多个 Glyph。
- 分享卡不得使用红章、卦象、裂纹、上涨箭头或收益感图形。

## 尺寸规则

| 场景 | 建议资产 | 视觉目标 |
| --- | --- | --- |
| 16px favicon | C16 | 只保留品牌残影 |
| 24px UI 角标 | C16 / UI Glyph | 清楚但不抢信息 |
| 32-40px 导航 | A1 / C16 | 能读出“照”的骨架 |
| 48-72px 报告落款 | A1 | 正式、克制、可信 |
| 海报封面 | A1 或精神符号 | 只出现一次，留足安全区 |

## 动效规则

- 允许：轻微淡入、边缘呼吸、极弱金色流光。
- 禁止：夸张弹跳、频繁闪烁、粒子爆发、赛博扫描线。
- Logo 动效只表达“照见”和“呼吸”，不表达“交易机会”。
- 动效必须可被 `prefers-reduced-motion` 降级。

## 可访问性规则

- 信息性 SVG 使用 `role="img"` 和 `aria-label`。
- 装饰性背景不需要读屏朗读。
- 小程序 CSS 几何标识使用 `aria-label`。
- 图片截图必须说明用途，不写“图片”两个字本身。
- 文本锁定能用真实文字就不用图片文字。

## 禁止

- 不把 A1 当页面水印铺满背景。
- 不把 C16 当按钮图标到处复制。
- 不用 UI Glyph 替代 Lucide 等通用操作图标。
- 不把“照”做成红印章、符咒、卦象或金融信号。
- 不把任何符号用于暗示买入、卖出、预测、收益或胜率。

## 当前判断

首页、报告、训练、小程序要先建立稳定识别，不急着堆满品牌资产。

好的标识系统不是让用户看见很多 Logo，而是让用户在不同入口都知道：这里是同一套“以交易照人心”的修行系统。
