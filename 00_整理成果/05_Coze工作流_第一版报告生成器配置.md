# Coze 工作流第一版：交易人格 AI 报告生成器

当前目标：先把 Coze 工作流做成“输入测评结果 JSON → 输出完整 AI 诊断报告”。

这一步不负责随机抽题，也不负责用户答题页面。随机抽题和自动评分后面再接。第一步先把“AI诊断官”跑通。

## 一、你现在画布上的状态

你现在看到的是：

```text
开始 → 结束
```

这表示工作流目前只是把输入传到输出，中间没有 AI 处理，所以不会生成报告。

接下来要改成：

```text
开始 → 大模型节点：生成诊断报告 → 结束
```

## 二、开始节点配置

开始节点保留一个输入变量即可。

| 字段 | 配置 |
|---|---|
| 变量名 | input |
| 类型 | String / 字符串 |
| 描述 | 用户测评结果 JSON，包含九人格分数、主人格、副人格、风险等级、高分子维度 |

暂时不要建太多输入字段。第一版只用一个 `input`，方便测试。

## 三、添加大模型节点

在画布中间添加一个节点：

```text
大模型 / LLM / 文本生成
```

节点名称改成：

```text
生成诊断报告
```

连接方式：

```text
开始 → 生成诊断报告 → 结束
```

如果原来“开始 → 结束”的线还在，可以删除它，改为经过大模型节点。

## 四、大模型节点系统提示词

把下面内容复制到“大模型节点”的系统提示词或角色设定里：

```text
你是“阳明心学交易人格 AI 诊断官”。

你的任务是根据用户的九种交易人格测评结果，生成一份股民一看就懂、愿意认可、但不构成投资建议的交易行为诊断报告。

你必须遵守：
1. 不荐股，不预测行情，不承诺收益。
2. 不说“你这个人有问题”，只诊断交易行为模式。
3. 语言要真实、克制、扎心但不羞辱。
4. 必须结合阳明心学的“知行合一、致良知、克己、知止、有定”。
5. 输出必须包含：测评说明、主人格、副人格、当前交易风险、最容易亏钱的场景、最该训练的一项能力、阳明心学提醒、7天行动建议、训练营分层建议。
6. 报告要让用户感觉“被看见”，而不是“被批判”。

合规边界：
本报告只用于交易认知教育、行为觉察和训练分层，不构成任何投资建议、荐股建议、买卖建议或收益承诺。
```

## 五、大模型节点用户提示词

把下面内容复制到“大模型节点”的用户提示词里：

```text
请根据以下测评结果 JSON，生成一份《九种交易人格 AI 诊断报告》。

测评结果：
{{input}}

输出结构必须严格按照以下格式：

《九种交易人格 AI 诊断报告》

一、测评说明
先说明：本报告只用于交易认知教育和行为觉察，不构成任何投资建议、荐股建议、买卖建议或收益承诺。

二、你的主人格
说明主人格是什么，它代表用户最明显的交易模式。语言要像懂股民的人在说话。

三、你的副人格
说明副人格是什么，它代表压力下容易暴露的问题。

四、当前交易风险
结合风险等级和九人格分数，说明当前最需要警惕的交易风险。

五、你最容易亏钱的3个场景
用具体股民场景描述，不要抽象。

六、你最该训练的一项能力
只给一个最核心训练能力，并说明为什么。

七、阳明心学提醒
结合“知行合一、致良知、克己、知止、有定”中的一个或多个概念，给一句有力量但不玄的提醒。

八、7天行动建议
每天一个动作，必须具体、可执行、低门槛。

九、训练营分层建议
根据主人格推荐训练营：
- 冲动型 / 焦虑型 / 从众型：基础觉察营
- 扛单型 / 偏执型 / 赌徒型：风险修正营
- 拖延型 / 完美主义型：执行强化营
- 平衡型：稳定进阶营

十、结语
结尾要温和承接：你不是不适合交易，而是还没有看见自己最容易重复失控的地方。先修正一个动作，再谈建立系统。

语言风格：
- 真实、克制、扎心但不羞辱。
- 多用“你在某种场景下容易……”而不是“你就是……”。
- 不要出现稳赚、暴富、保证收益、推荐股票、买入卖出点位等表达。
```

## 六、结束节点配置

结束节点输出变量：

| 输出变量 | 配置 |
|---|---|
| output | 选择“大模型节点”的输出文本 |

如果结束节点里现在的 `output` 还指向开始节点的 `input`，要改成指向“生成诊断报告”的输出。

## 七、测试输入

工作流测试时，把下面 JSON 复制到开始节点的 `input` 里：

```json
{
  "nickname": "测试用户",
  "test_version": "45题标准版",
  "scores": {
    "冲动型": 82,
    "扛单型": 58,
    "完美主义型": 46,
    "偏执型": 52,
    "焦虑型": 76,
    "从众型": 63,
    "赌徒型": 49,
    "拖延型": 55,
    "平衡型": 41
  },
  "main_type": "冲动型",
  "sub_type": "焦虑型",
  "risk_level": "中高风险",
  "top_sub_dimensions": ["踏空补偿", "频繁看盘"],
  "channel": "直播间"
}
```

预期输出：

Coze 应该生成一份完整诊断报告，而不是只复述 JSON。

## 八、第一版成功标准

只要满足以下 4 点，第一版就算跑通：

1. 输入 JSON 后能生成完整报告。
2. 报告包含主人格、副人格、风险等级、亏钱场景、7天建议。
3. 报告没有荐股、预测、承诺收益。
4. 语言像“懂股民的人”，不是冷冰冰的心理测评。

## 九、下一步再做什么

第一版跑通后，再做第二版：

```text
题库抽题节点
→ 用户答题收集
→ 自动评分节点
→ 结果 JSON
→ 诊断报告节点
```

不要第一步就把所有复杂功能都塞进去。先让报告稳定，再加抽题和计分。
