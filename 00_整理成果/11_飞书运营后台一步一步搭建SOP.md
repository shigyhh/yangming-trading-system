# 飞书运营后台一步一步搭建 SOP

适用项目：AI交易人格测评系统 2.0  
目标：把飞书搭成“线索承接 + 报告沉淀 + 助理跟进 + 数据看板”的轻量运营后台。  
原则：第一版先用飞书多维表格 + 飞书群机器人跑通运营闭环；正式版再由后端服务写入飞书。

参考官方文档：

- 飞书多维表格 OpenAPI 概述：https://open.feishu.cn/document/server-docs/docs/bitable-v1/bitable-overview
- 飞书多维表格新增记录 API：https://open.feishu.cn/document/server-docs/docs/bitable-v1/app-table-record/create?lang=zh-CN
- 飞书自定义机器人使用指南：https://open.feishu.cn/document/client-docs/bot-v3/add-custom-bot

---

## 一、最终要搭出来什么

飞书里建议建 1 个多维表格应用：

```text
AI交易人格测评运营台
```

里面建 5 张表：

```text
01_用户线索
02_测评记录
03_答案明细
04_AI报告
05_运营配置
```

再建 1 个飞书群：

```text
AI交易人格测评线索群
```

群里加 1 个自定义机器人：

```text
交易人格线索通知机器人
```

用户完成测评后的数据流：

```text
网页/后端
→ 飞书多维表格写入用户、测评、报告
→ 飞书群机器人推送新线索提醒
→ 助理在飞书里跟进
```

---

## 二、第一步：创建飞书多维表格

### 1. 操作路径

1. 打开飞书。
2. 新建。
3. 选择「多维表格」。
4. 命名为：

```text
AI交易人格测评运营台
```

### 2. 建表顺序

先建这 5 张表：

```text
01_用户线索
02_测评记录
03_答案明细
04_AI报告
05_运营配置
```

## 三、第二步：配置 01_用户线索 表

| 字段名 | 字段类型 | 是否必填 | 说明 |

| 手机号 | 电话号码/文本 | 否 | 可脱敏展示 |
| 微信绑定 | 单选 | 否 | 已绑定 / 未绑定 |
| 微信标识 | 文本 | 否 | openid/unionid，内部用 |
| 来源渠道 | 单选 | 是 | 直播间 / 短视频 / 私域社群 / 体验课 / 线下沙龙 |
| 邀请码 | 文本 | 否 | 渠道码 |
| 助理状态 | 单选 | 是 | 未添加 / 已添加 / 待确认 / 已进群 |
| 最近主人格 | 单选 | 否 | 九种人格 |
| 最近风险等级 | 单选 | 否 | 高风险 / 中高风险 / 中等风险 / 低风险 |
| 推荐训练营 | 单选 | 否 | 基础觉察营 / 风险修正营 / 执行强化营 / 稳定进阶营 |
| 跟进状态 | 单选 | 是 | 待跟进 / 已联系 / 已进群 / 已报名 / 暂不跟进 |
| 助理备注 | 多行文本 | 否 | 人工跟进记录 |
| 注册时间 | 日期时间 | 是 | 首次进入系统时间 |
| 最近测评时间 | 日期时间 | 否 | 最近一次测评完成时间 |

### 推荐视图

1. 今日新增线索

| 主人格 | 单选 | 否 | 九种人格 |
| 副人格 | 单选 | 否 | 九种人格 |

## 五、第四步：配置 03_答案明细 表

用途：保存每个用户每一道题的答案，用于后续统计与模型优化。

| 字段名 | 字段类型 | 是否必填 | 说明 |
|---|---|---|---|
| 答案ID | 文本 | 是 | answer_id |
| 测评ID | 文本 | 是 | assessment_id |
| 用户ID | 文本 | 是 | user_id |
| 题目ID | 文本 | 是 | question_id |
| 人格类型 | 单选 | 是 | 九种人格 |
| 子维度 | 文本/单选 | 是 | 追涨冲动等 |
| 分数 | 数字 | 是 | 1-5 |
| 权重 | 数字 | 否 | 默认 1 |
| 作答时间 | 日期时间 | 是 | answered_at |

### 注意

正式版如果数据量大，答案明细不一定全部同步到飞书，可以只进数据库。飞书第一版可以先同步测评汇总和报告，不同步 45 条答案明细，避免表格太重。

## 九、第八步：添加飞书自定义机器人

### 操作路径

1. 进入「AI交易人格测评线索群」。
2. 点击群设置。
3. 点击群机器人。
4. 添加机器人。
5. 选择「自定义机器人」。
6. 名称：

```text
交易人格线索通知机器人
```

7. 复制 Webhook 地址。

Webhook 格式类似：

```text
https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx
```

### 安全设置

建议至少开启一种安全设置：

1. 签名校验：推荐。
2. IP 白名单：正式后端服务器 IP 固定后再开启。
3. 自定义关键词：可作为补充。

飞书官方说明：自定义机器人支持通过 Webhook 推送消息，并支持关键词、IP 白名单、签名校验等安全配置。自定义机器人频率限制为单租户单机器人 100 次/分钟、5 次/秒，请避免短时间批量推送。

### 测试消息

请求体：

```json
{
  "msg_type": "text",
  "content": {
    "text": "交易人格测评线索机器人测试成功"
  }
}
```

## 十、第九步：创建飞书开放平台应用

用途：让后端可以把数据写入多维表格。

### 操作路径

1. 打开飞书开放平台。
2. 进入开发者后台。
3. 创建企业自建应用。
4. 应用名：

```text
AI交易人格测评系统
```

5. 记录：

```text
FEISHU_APP_ID
FEISHU_APP_SECRET
```

### 权限建议

申请多维表格相关权限：

1. 查看多维表格
2. 编辑多维表格
3. 新增记录
4. 查看记录
5. 更新记录

具体权限名称以后端实际调用 API 时飞书后台提示为准。

### 关键授权

飞书官方文档强调：调用多维表格 API 前，应用需要是多维表格所有者或协作者，否则可能返回 403 或 400。

所以必须做：

1. 打开「AI交易人格测评运营台」多维表格。
2. 点击分享/协作。
3. 把你的企业自建应用添加为协作者。
4. 给编辑权限。

## 十一、第十步：记录 app_token 和 table_id

### app_token

多维表格 URL 里可以看到 `app_token`。

常见 URL 形态：

```text
https://xxx.feishu.cn/base/xxxxxx
```

其中 `base/` 后面的部分就是多维表格 app token。

### table_id

打开某张表时，URL 里通常能看到 table 参数。也可以通过飞书 API 获取表列表。

建议记录：

```env
FEISHU_BITABLE_APP_TOKEN=
FEISHU_TABLE_USERS_ID=
FEISHU_TABLE_ASSESSMENTS_ID=
FEISHU_TABLE_ANSWERS_ID=
FEISHU_TABLE_REPORTS_ID=
FEISHU_TABLE_CONFIG_ID=
FEISHU_BOT_WEBHOOK=
FEISHU_BOT_SECRET=
```

## 十二、第一版同步策略

第一版建议先同步 3 类数据：

### 1. 用户线索

写入 `01_用户线索`。

触发时机：

```text
用户登录成功
```

### 2. 测评记录

写入 `02_测评记录`。

触发时机：

```text
用户完成测评并生成评分结果
```

### 3. AI 报告

写入 `04_AI报告`。

触发时机：

```text
AI报告生成成功
```

暂时不建议第一版同步完整答案明细，除非你要做题目质量分析。

3. 命名为 `AI交易人格测评运营台`。
4. 建第一张表 `01_用户线索`。
