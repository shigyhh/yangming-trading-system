# 阳明心学交易修行系统：数据模型文档

## 设计原则

后端数据模型服务于“交易修行闭环”，不是服务于交易执行。

核心数据边界：

- 保存修行状态，不保存具体交易明细。
- 保存心境、戒律、训练、复盘、成长树、道场关系。
- 教练/助教只看闭环状态，不看具体股票、买卖点、账户盈亏。
- 当前本地开发使用 `server/data/runtime/*.json`，正式版可迁移到 `server/src/db/schema.sql` 中的 PostgreSQL 表结构。

## 模块总览

| 模块 | 数据 | 用途 |
| --- | --- | --- |
| 用户与身份 | `users`、`auth_sessions`、`auth_identities`、`sms_codes` | 注册登录、小程序同步、邀请码、来源追踪 |
| 测评与档案 | `assessment_sessions`、`assessment_answers`、`score_results`、`reports` | 九面行为心镜、主副心镜、训练路径、档案 |
| 每日修行 | `daily_checkins`、`miniprogram-states.json`、`mind_profile`、`mind_records`、`training_state`、`review_records`、`zhixing_score` | 照心、修行、复盘、连续修行、知行指数 |
| 活镜成长 | `mirror_reports`、`training_records`、`trade_reviews`、`living_mirror_stats`、`assistant_handoffs` | 心镜报告、训练证据、真实交易复盘、活镜成长、助教承接 |
| 内容库 | `content365` | 每日心证、戒律、训练、复盘，支撑小程序、直播、视频号和AI观心教练 |
| K线训练 | `kline_practice_bank`、`kline_practice_results`、知行逐根回放运行数据 | 匿名历史K线、逐根回放、知行合一训练 |
| 修行道场 | `dojo_mentors`、`dojo_mentor_bindings`、`dojo_tasks`、`dojo_task_records`、`dojo_mind_records` | 教练/助教、绑定关系、共修任务、观心记录、修行榜 |

## LivingMirror v1

Sprint 9 之后以「活镜」作为 Web、小程序、后台、server 的统一业务结构。共享契约见 `packages/contracts/living-mirror.d.ts`。

核心实体：

- `User`：一个手机号一个用户。
- `MirrorReport`：心镜报告，包含主镜、副镜、心贼、判词、风险雷达、典型循环、7 天修行处方。
- `TrainingRecord`：每日训练记录，记录日期、心镜、训练动作、完成状态和省察。
- `TradeReview`：真实交易复盘 MVP，记录截图地址、用户自述、最强念头、映射心镜和复盘文字。
- `LivingMirrorStats`：活镜成长数据，聚合九镜强度、心贼频次、训练完成率、循环复发次数和良知之镜成长值。
- `AssistantHandoff`：助教承接摘要，记录最近主副心镜、风险标签、复盘摘要、建议训练动作和合规提醒。

九面行为心镜：

- 追涨之镜
- 扛单之镜
- 幻想之镜
- 赌性之镜
- 从众之镜
- 犹疑之镜
- 拖延之镜
- 焦虑之镜
- 良知之镜

规则：

- 每次训练、复盘、上传截图，都应影响 `LivingMirrorStats`。
- 截图只作为复盘证据，不作为行情判断依据。
- 助教只看必要摘要和风险标签，不读取敏感截图细节。
- Web 负责深度照见，小程序负责高频记录，后台负责承接和运营。
- 所有输出只用于交易心理觉察、行为训练与复盘，不构成投资建议。

## User

用户是系统中的修行者，不等于交易账户。

核心字段：

- `id`
- `nickname`
- `phone`
- `personal_invite_code`
- `source_channel`
- `created_at`

规则：

- 用户数据只用于修行闭环、训练营承接和合规授权。
- 不保存券商账户、持仓、收益截图等交易明细。

## PersonalityProfile

人格档案来自九型人格测评。

核心字段：

- `main_type`
- `sub_type`
- `risk_level`
- `score_percentages`
- `actions_7_days`
- `yangming_reminder`

九型枚举：

- 冲动型
- 扛单型
- 完美型
- 赌徒型
- 从众型
- 偏执型
- 拖延型
- 焦虑型
- 平衡型

规则：

- 人格档案只用于训练分层。
- 不能输出投资建议。
- 动态档案后续应叠加最近7天行为证据、常破戒律、知行指数趋势和教练备注。

## MindRecord

照心记录用于记录开盘前或观心时的心境。

核心字段：

- `user_id`
- `date_key`
- `mind_key`
- `mind_name`
- `one_thing`
- `discipline`
- `index_focus`
- `created_at`

心境枚举：

- 急
- 贪
- 惧
- 静
- 怒
- 疲
- 赌
- 空

规则：

- 照心不是问卷。
- 照心生成今日戒律、今日禁忌、今日只练一件事、今日知行提醒。

## TrainingState

每日修行训练状态。

核心字段：

- `date_key`
- `steps.trigger`
- `steps.micro`
- `steps.reflection`
- `reflection`
- `completed`
- `mindTask`
- `indexFocus`
- `trainingCard`

`trainingCard` V1 字段：

- `id`
- `dateKey`
- `personalityType`
- `stageKey`
- `stageName`
- `heartThief`
- `commandment`
- `trigger`
- `action`
- `review`
- `contentDayId`
- `heartProof`

规则：

- 每天只练一件事。
- 完成条件是三步闭环：照见触发、完成微训练、写下省察。
- 完成后参与连续修行计算。

## ReviewRecord

复盘记录用于生成 AI省察、AI复盘和知行指数。

核心字段：

- `date_key`
- `traded`
- `inSystem`
- `chased`
- `held`
- `changedPlan`
- `executedStop`
- `revenge`
- `emotion`
- `score`
- `correction`
- `aiReflection`
- `aiReplay`

规则：

- 复盘不评价结果，只检查知与行是否一致。
- 字段不得出现具体股票、收益、买卖建议。

## ZhixingIndex

知行指数是聚合指标，不单独作为交易评价。

维度：

- 纪律执行：权重 26%
- 情绪稳定：权重 20%
- 系统一致性：权重 22%
- 临盘克制：权重 16%
- 主修兑现：权重 16%

输入：

- 九型人格档案
- 主修关卡
- 今日照心
- 每日训练卡
- 心性复盘

规则：

- 总分范围：20 到 98。
- 不允许与收益、交易结果、具体股票绑定。
- 每日写入 `zhixing_score.records[date_key]`，用于近7天、近30天趋势。
- `zhixing_score.latest` 只保存当前最新知行指数快照。
- 算法输入只来自九型人格、主修关卡、今日照心、每日训练卡、心性复盘。
- 记录快照必须包含 `personality`、`mainStage`、`growthBinding`，用于证明指数不是孤立数字，而是人格与关卡成长的结果。

## MindProfile

`mind_profile` 是小程序内的当前修行快照，用于连接人格、关卡、知行指数和今日训练。

核心字段：

- `personality_type`
- `current_stage`
- `current_stage_key`
- `heart_thief`
- `zhixing_score`
- `discipline_rate`
- `reflection_count`
- `streak_days`
- `current_mind_state`
- `today_training`
- `today_commandment`

规则：

- 该模型不保存具体交易明细。
- 任何页面更新人格、照心、训练、复盘、知行指数时，都可以刷新该快照。
- 训练营教练/助教只读取这个快照和闭环状态，不读取交易标的。

## StageSystem

六大关卡是当前产品主轴。

关卡枚举：

- 立志
- 照心
- 事上磨
- 破心贼
- 知行合一
- 致良知

输入：

- 立志承诺
- 照心记录
- 九型档案
- 今日训练
- AI复盘
- 知行指数
- 连续修行
- 道场共修任务

规则：

- 关卡进度只体现修行闭环，不体现收益。
- 道场任务完成应反哺事上磨关和致良知关。
- 人格测评结果必须映射到当前主修关卡、当前心贼、今日戒律和今日训练。
- 每关必须定义 `core`、`goal`、`trainingFocus`、`unlockCriteria`、`completionEvidence`、`output`。
- 六关定义以 `worldview/six-stages.md` 为源文件，代码中的关卡元数据只能同步它，不能另起一套解释。

## GrowthLevel

`growth_level` 是用户长期修行等级。

等级枚举：

- 观己
- 守志
- 照心
- 事上磨
- 破贼
- 知行
- 守一
- 良知

输入：

- 连续修行天数
- 知行指数
- 完成任务数
- 省察记录数

规则：

- 等级只表达修行进度，不表达交易能力。
- 等级可用于个人页、成长记录页、训练营运营看板。

## ContinuityState

连续修行状态。

核心字段：

- `currentStreak`
- `bestStreak`
- `recentDays`
- `todayProgress`
- `milestone`

里程碑：

- 7日连修
- 21日连修
- 49日连修
- 108日连修

规则：

- 训练完成才算连修。
- 照心、训练、复盘共同组成每日闭环进度。

## DojoMentor

真人带练身份。

表：`dojo_mentors`

核心字段：

- `id`
- `user_id`
- `role`
- `mentor_code`
- `display_name`
- `bio`
- `status`

角色枚举：

- `coach`：训练营教练
- `assistant`：助教

规则：

- 不使用“老师”作为真人角色主名称。
- 真人带练只做修行陪跑和监督。

## DojoMentorBinding

学员与真人带练的绑定关系。

表：`dojo_mentor_bindings`

核心字段：

- `user_id`
- `mentor_id`
- `mentor_user_id`
- `mentor_code`
- `role`
- `status`

状态：

- `active`
- `replaced`
- `revoked`

规则：

- 一个用户同一时间只有一个 active 真人带练关系。
- 用户不能绑定自己。
- 绑定后教练/助教只能看修行闭环状态。

## DojoTask

道场共修任务。

表：`dojo_tasks`

核心字段：

- `creator_user_id`
- `title`
- `discipline`
- `action`
- `personality_type`
- `stage`
- `target_role`
- `status`

规则：

- 任务必须对齐九型人格或六大关卡。
- 任务动作必须可在 1 到 3 分钟内完成。
- 任务不得包含交易指令。

## DojoTaskRecord

用户领取和完成道场任务的记录。

表：`dojo_task_records`

核心字段：

- `user_id`
- `task_id`
- `date_key`
- `status`
- `note`
- `accepted_at`
- `completed_at`

状态：

- `accepted`
- `completed`

规则：

- 完成任务后反哺成长树。
- 完成任务可参与修行榜。

## DojoMindRecord

AI观心助手记录。

表：`dojo_mind_records`

核心字段：

- `user_id`
- `date_key`
- `input`
- `reply`
- `context`
- `created_at`

上下文应包含：

- `personality_type`
- `mind`
- `training`
- `review`
- `growth_gate`
- `continuity`
- `mentor_binding`

规则：

- 观心记录只保存修行上下文。
- 不保存具体股票和交易明细。
- 真实大模型接入时，生成内容也必须遵守世界观文档。

## CoachModule

`modules/coach` 是 AI观心教练预留接口，暂不接真实模型。

输入：

- `user_emotion`
- `user_review`
- `user_question`
- `personality_type`
- `current_stage`
- `heart_thief`
- `today_training`
- `today_commandment`

输出：

- `trigger_personality`
- `current_heart_thief`
- `suggestion`
- `today_only_practice`
- `commandment_reminder`
- `coach_note`

规则：

- 输出只能是观心、戒律、训练动作。
- 不输出买卖、标的、仓位指令或收益承诺。

## Content365

`modules/content365` 是 365 天修行内容库。

每一天包含：

- `id`：例如 `Day001`
- `title`：今日心证
- `stageKey`
- `stageName`
- `stageCore`
- `heartProof`
- `commandment`
- `training`
- `review`
- `liveTopic`

规则：

- 内容必须绑定六大关卡。
- 每条内容必须包含戒律、心证、训练、复盘。
- 内容可用于小程序首页、365内容库页面、直播提纲、视频号口播和AI观心教练。

## API 模型

### 用户道场概览

`GET /api/v1/users/:user_id/dojo/summary`

返回：

- 道统导师
- 真人带练关系
- 下一条共修任务
- 任务统计
- 最近观心记录
- 修行榜位置

### 绑定训练营教练/助教

`POST /api/v1/users/:user_id/dojo/bindings`

请求：

```json
{
  "mentor_code": "ZXABC123",
  "role": "coach"
}
```

### 注册训练营教练/助教

`POST /api/v1/dojo/mentors/register`

请求：

```json
{
  "role": "coach",
  "display_name": "老何训练营教练",
  "bio": "只做修行陪跑，不做交易建议"
}
```

### 道场任务

`GET /api/v1/dojo/tasks`

`POST /api/v1/dojo/tasks`

### 用户任务记录

`POST /api/v1/users/:user_id/dojo/tasks/:task_id/records`

请求：

```json
{
  "action": "complete",
  "note": "今天完成行动前三问"
}
```

### 观心记录

`GET /api/v1/users/:user_id/dojo/mind-records`

`POST /api/v1/users/:user_id/dojo/mind-records`

### 教练/助教看板

`GET /api/v1/dojo/mentor-dashboard`

返回：

- 学员总数
- 需要跟进人数
- 今日照心状态
- 今日训练状态
- 今日复盘状态
- 今日共修任务状态

### 修行榜

`GET /api/v1/dojo/leaderboard?period=week|month|all`

规则：

- 只按修行行为计分。
- 不允许加入收益、账户金额、具体标的表现。
