# 阳明心学交易修行系统：数据模型文档

## 设计原则

后端数据模型服务于“交易修行闭环”，不是服务于交易执行。

核心数据边界：

- 保存修行状态，不保存具体交易明细。
- 保存心境、戒律、训练、复盘、成长树、道场关系。
- 教练/助教只看闭环状态，不看具体股票、买卖点、账户盈亏。
- 当前本地开发使用 `server/data/runtime/*.json`，正式版可迁移到 `server/src/db/schema.sql` 中的 PostgreSQL 表结构。

## 模块总览

| 模块 | 数据 | 用途 |
| --- | --- | --- |
| 用户与身份 | `users`、`auth_sessions`、`auth_identities`、`sms_codes` | 注册登录、小程序同步、邀请码、来源追踪 |
| 测评与档案 | `assessment_sessions`、`assessment_answers`、`score_results`、`reports` | 九型人格、主次人格、训练路径、档案 |
| 心镜生命体闭环 | `mirror_reports`、`trade_reviews`、`personal_cycles`、`daily_reflections`、`training_records`、`living_mirror_stats`、`living_mirror_events`、`mirror_scroll_entries` | 入照心、心镜报告、真实交易复盘、今日心证、心镜之树、心镜长卷 |
| 每日修行 | `daily_checkins`、`miniprogram-states.json`、`mind_profile`、`mind_records`、`training_state`、`review_records`、`zhixing_score` | 照心、修行、复盘、连续修行、知行指数 |
| 内容库 | `content365` | 每日心证、戒律、训练、复盘，支撑小程序、直播、视频号和AI观心教练 |
| K线训练 | `kline_practice_bank`、`kline_practice_results`、知行逐根回放运行数据 | 匿名历史K线、逐根回放、知行合一训练 |
| 修行道场 | `dojo_mentors`、`dojo_mentor_bindings`、`dojo_tasks`、`dojo_task_records`、`dojo_mind_records` | 教练/助教、绑定关系、共修任务、观心记录、修行榜 |

## User

用户是系统中的修行者，不等于交易账户。

核心字段：

- `id`
- `nickname`
- `phone`
- `personal_invite_code`
- `source_channel`
- `created_at`

规则：

- 用户数据只用于修行闭环、训练营承接和合规授权。
- 不保存券商账户、持仓、收益截图等交易明细。

核心字段：

- `main_type`
- `sub_type`
- `risk_level`
- `score_percentages`
- `actions_7_days`
- `yangming_reminder`

- 偏执型

- 人格档案只用于训练分层。
- 不能输出投资建议。
- 动态档案后续应叠加最近7天行为证据、常破戒律、知行指数趋势和教练备注。

## MirrorReport

`mirror_reports` 是 Sprint 9 的心镜报告，不是普通人格报告。

核心字段：

- `user_id`
- `assessment_id`
- `mirror_spectrum`
- `main_mirror`
- `sub_mirror`
- `primary_mirror_score`
- `secondary_mirror_score`
- `thieves`
- `verdict`
- `risk_radar`
- `typical_cycle`
- `seven_day_prescription`
- `camp_suggestion`
- `compliance_note`

规则：

- 主镜、副镜、心贼来自入照心与九面行为心镜，但前端优先展示「镜谱」。
- `main_mirror` 和 `sub_mirror` 只是当前峰值，不把用户固化为某种标签。
- 报告必须输出今日修行动作，但不能输出买卖建议。
- `typical_cycle` 使用真实交易语言，例如“怕错过 -> 追涨 -> 被套 -> 不甘 -> 再次追涨”。
- 报告生成后必须写入 `living_mirror_events`，作为心镜生命体的第一条证据。

## TradeReview

`trade_reviews` 是 Sprint 10.5 的真实交易复盘 MVP。

第一版只做：

- 截图上传
- 用户自述
- 心镜映射
- 复盘记录

核心字段：

- `user_id`
- `image_url`
- `image_storage_key`
- `trade_date`
- `symbol_masked`
- `market_type`
- `buy_reason`
- `sell_reason`
- `strongest_thought`
- `detected_mirror`
- `detected_thieves`
- `behavior_tags`
- `personal_cycle`
- `review_text`
- `ai_summary`

映射规则：

- 怕错过 -> 追涨之镜
- 想翻本 -> 赌性之镜
- 不认错 -> 扛单之镜
- 赚一点怕回吐 -> 焦虑之镜
- 大家都在说 -> 从众之镜

规则：

- 不追求第一版完全自动识别行情。
- 可以保存脱敏品种字段 `symbol_masked`，不要保存完整持仓、账户、盈亏。
- 复盘文本只讲行为、念头、训练，不讲具体买卖。
- 每条复盘必须写入 `living_mirror_events`，用于刷新心镜生命体。
- 每条复盘都应尝试沉淀或更新 `personal_cycles`，让循环之镜展示“我的循环”。

## PersonalCycle

`personal_cycles` 是用户自己的循环之镜数据。

核心字段：

- `user_id`
- `mirror`
- `trigger_text`
- `thought_text`
- `action_text`
- `result_text`
- `recurrence_text`
- `thieves`
- `evidence_review_id`
- `occurrence_count`
- `last_occurred_at`

示例：

```text
触发：群里都在说
念头：再不上车来不及了
动作：追涨
结果：被套
再次触发：更怕错过
```

规则：

- 循环之镜优先读取 `personal_cycles`，不要永远展示系统演示循环。
- 循环内容必须使用真实交易语言。
- 不记录完整账户、收益、持仓。

## DailyReflection

`daily_reflections` 是 Sprint 10.8 的今日心证。

它不是签到，而是每天一次「今日照见 + 今日修行 + 心镜之树生长」。

核心字段：

- `user_id`
- `date_key`
- `mirror`
- `thieves`
- `today_insight`
- `practice_action`
- `completed`
- `completed_at`
- `source_channel`

示例：

```text
今日照见：
你追的不是机会，是怕错过。

今日修行：
今天没有系统信号，不主动找机会。
```

规则：

- 每天只给一件可执行修行动作。
- 完成后写入 `living_mirror_events`。
- 完成后推动 `living_mirror_stats.tree_state` 与 `vitality_score`。

- 训练不是签到，而是心镜生命体成长证据。
- 每次完成训练，都要影响 `living_mirror_stats`。
- 每天只练一件事，动作必须可执行。

## LivingMirrorStats

`living_mirror_stats` 是整个产品的核心资产，即「心镜生命体」。

它既保存可计算的镜谱，也保存可视化的心镜之树状态。

命名规则：

- 用户侧默认展示为「心镜之树」。
- 连续修行、复盘、落印后，可在高阶状态显影为「良知之树」。
- 这棵树属于用户，也属于《照见》活镜成长系统，不是普通积分或等级。

核心字段：

- `user_id`
- `mirror_spectrum`
- `mirror_scores`
- `thief_counts`
- `life_form_type`
- `life_display_name`
- `life_stage`
- `vitality_score`
- `tree_state`
- `growth_trend`
- `training_completion_rate`
- `cycle_relapse_count`
- `conscience_growth`
- `last_main_mirror`
- `last_sub_mirror`
- `last_updated`

输入：

- 心镜报告
- 今日心证
- 训练记录
- 真实交易复盘
- 循环之镜复发记录
- 复测变化

输出示例：

```text
追涨之镜 ↓ 12
焦虑之镜 ↓ 8
良知之镜 ↑ 15

心镜之树：幼芽长出第二片叶。
```

规则：

- 心镜生命体不是投资能力评分。
- 指标只表达交易心理、行为惯性与训练变化。
- `tree_state` 应记录树的阶段、枝叶、光泽、根系、落印次数等表现数据。
- 训练、复盘、截图三问、今日心证都可以让心镜之树成长。
- 网页、小程序、后续 App 必须读取同一份 `living_mirror_stats`。

## ZhixingIndex

知行指数是聚合指标，不单独作为交易评价。

维度：

- 纪律执行：权重 26%
- 情绪稳定：权重 20%
- 系统一致性：权重 22%
- 临盘克制：权重 16%
- 主修兑现：权重 16%

输入：

- 九型人格档案
- 主修关卡
- 今日照心
- 每日训练卡
- 心性复盘

规则：

- 总分范围：20 到 98。
- 不允许与收益、交易结果、具体股票绑定。
- 每日写入 `zhixing_score.records[date_key]`，用于近7天、近30天趋势。
- `zhixing_score.latest` 只保存当前最新知行指数快照。
- 算法输入只来自九型人格、主修关卡、今日照心、每日训练卡、心性复盘。
- 记录快照必须包含 `personality`、`mainStage`、`growthBinding`，用于证明指数不是孤立数字，而是人格与关卡成长的结果。

`modules/coach` 是 AI观心教练预留接口，暂不接真实模型。

输入：

- `user_emotion`
- `user_review`
- `user_question`
- `personality_type`
- `current_stage`
- `heart_thief`
- `today_training`
- `today_commandment`

输出：

- `trigger_personality`
- `current_heart_thief`
- `suggestion`
- `today_only_practice`
- `commandment_reminder`
- `coach_note`

规则：

- 输出只能是观心、戒律、训练动作。
- 不输出买卖、标的、仓位指令或收益承诺。

## Content365

`modules/content365` 是 365 天修行内容库。

每一天包含：

- `id`：例如 `Day001`
- `title`：今日心证
- `stageKey`
- `stageName`
- `stageCore`
- `heartProof`
- `commandment`
- `training`
- `review`
- `liveTopic`

规则：

- 内容必须绑定六大关卡。
- 每条内容必须包含戒律、心证、训练、复盘。
- 内容可用于小程序首页、365内容库页面、直播提纲、视频号口播和AI观心教练。

## API 模型

### 用户道场概览

`GET /api/v1/users/:user_id/dojo/summary`

返回：

- 道统导师
- 真人带练关系
- 下一条共修任务
- 任务统计
- 最近观心记录
- 修行榜位置

### 绑定训练营教练/助教

`POST /api/v1/users/:user_id/dojo/bindings`

请求：

```json
{
  "mentor_code": "ZXABC123",
  "role": "coach"
}
```

### 注册训练营教练/助教

`POST /api/v1/dojo/mentors/register`

请求：

```json
{
  "role": "coach",
  "display_name": "老何训练营教练",
  "bio": "只做修行陪跑，不做交易建议"
}
```

### 道场任务

`GET /api/v1/dojo/tasks`

`POST /api/v1/dojo/tasks`

### 用户任务记录

`POST /api/v1/users/:user_id/dojo/tasks/:task_id/records`

请求：

```json
{
  "action": "complete",
  "note": "今天完成行动前三问"
}
```

### 观心记录

`GET /api/v1/users/:user_id/dojo/mind-records`

`POST /api/v1/users/:user_id/dojo/mind-records`

### 教练/助教看板

`GET /api/v1/dojo/mentor-dashboard`

返回：

- 学员总数
- 需要跟进人数
- 今日照心状态
- 今日训练状态
- 今日复盘状态
- 今日共修任务状态

### 修行榜

`GET /api/v1/dojo/leaderboard?period=week|month|all`

规则：

- 只按修行行为计分。
- 不允许加入收益、账户金额、具体标的表现。
